# STELLA - Concept Document

## Name Origin
**STELLA** stands for:
**S**tateful **T**urn-based **E**xecution & **L**LM **L**oop **A**rchitecture

（状態保持型・ターン制実行 および LLMループ・アーキテクチャ）

ラテン語で「星」を意味するこの名前は、暗闇（不確実なLLMの出力）の中で、プログラムによるループ制御が「道しるべ（星）」と なってタスクを完遂に導くイメージから名付けられました。「DIVA（歌姫）」をルーツに持つエージェントの系譜です。

## The Paradigm Shift: 概念の転換

従来のプロンプト・エンジニアリングでは、LLMは「関数（y = f(x)）」として扱われていました。入力を与えれば出力を返し、そこ で終了（Stateless）します。

STELLAは、この主従関係を逆転させます。
**「LLMをCPU（頭脳）としてループの中に閉じ込め、プログラムは環境（OS）に徹する」**

1. **入力場所の提供 (Eyes and Hands)**
   LLMには `run_shell` や `ask_user` といった「ツール（関数）」が与えられます。これにより、LLMは自ら環境に干渉し、結果を読み取ることができます。
2. **主導権の逆転 (Inversion of Control)**
   いつタスクを完了し、いつセッションを閉じるか（`finish`）の決定権は、人間でもプログラムでもなく、LLM自身が持ちます。
3. **対話の永続化 (Stateful Single Session)**
   ループが回っている間、コンテキストは保持され続けます。これにより「1回のリクエストで、AIと20回やり取りして、10個のバグを直させる」という究極の高効率・低コストな稼働が実現します。

## Core Value
- **1リクエストの極大化**: GitHub Copilot Premiumなどの「1リクエスト＝1カウント」という制約の中で、1枚のチケットで長時間の重労働をアサインできるビジネスモデルの確立。
- **自律的な軌道修正**: 実行して失敗したら自分で直し、分からなければ人間に `ask_user` で質問する。投げっぱなしにならない高い完遂率。

## Epilogue: 開発の終着点と価値
このプロジェクトは、「自己責任（Self-Responsibility）」で動作する自律ツールの最小構成として完成しました。
実用的なツールであると同時に、以下の価値を持つ「雛形」でもあります。

- **ミニマリズム**: 複雑なフレームワーク（LangChain等）に頼らず、標準的なライブラリのみで「LLMのループ駆動」を実現。
- **透明性**: APIとのやり取りを完全に制御下に置くことで、エージェントが「何を考え、何をしようとしているか」を常に把握可能。
- **拡張性**: ツール（関数）を追加するだけで、エージェントの能力を際限なく広げられる構造。

この「星」は、自律型エージェント開発の夜明けを照らす道しるべとなります。
